## 二、产品定位（继承用户 PRD）

- **一句话定位**: 无尽矿井是矿场增长游戏中的主动下挖模式：玩家消耗 `1` 点体力进入一局，系统固定发放 `20` 个木镐子作为基础探索资源，在深度推进和收益止盈的选择中，把本局产出稳定回流到主矿场资源体系。
- **模式身份**: 主矿场系统中的扩展玩法模式，不是独立单局游戏。
- **目标用户**: 缺失：用户 PRD 未明确目标用户画像；已明确体验场景为竖屏移动端短时主动操作体验。
- **平台**: 独立 H5，竖屏移动端为唯一主设计目标。
- **设计目标**:
  - 模式身份清晰：主矿场内扩展模式，不独立产生新经济体系。
  - 开局成本简单：`1` 点体力进入 `1` 局，单局固定 `20` 个木镐子。
  - 深挖收益明确：越深层收益期望越高。
  - 决策张力成立：玩家需要在继续下挖和及时止盈之间判断。
  - 资源模型统一：所有收益统一进入主矿场背包、钱包和材料系统。

## 三、系统设计

### 3.1 入口与单局开局系统

- **系统概述**: 负责从矿区主页进入无尽矿井，并完成体力消耗与本局基础工具发放。
- **核心流程**:
  1. 玩家在矿区主页点击「无尽矿井」入口。
  2. 系统检查体力。
  3. 体力足够时，消耗 `1` 点体力。
  4. 创建 `1` 局无尽矿井挑战。
  5. 系统发放本局固定 `20` 个木镐子。
  6. 进入无尽矿井页面，展示顶部 HUD、状态信息区、顶部收获累计区、中部 7×6 挖掘棋盘、右侧工具条、背包入口和「收队返场」。
- **规则定义**:
  - `1` 点体力 = `1` 局无尽矿井。
  - `1` 局无尽矿井 = 以固定 `20` 个木镐子开局的一次连续下挖挑战。
  - 体力不足时不可进入无尽矿井。
  - 本模式不额外创造门票或独立入场资源。
  - 木镐子开局数量长期稳定为 `20` 个。
- **资源关联**:
  - 消耗主矿场通用体力。
  - 顶部 HUD 延续矿区主页口径，展示体力、主货币、付费货币等主系统信息。
- **边界与异常**:
  - 体力不足时入口置灰并提示 `体力不足`。
  - 点击入口需设置防抖，入场处理中禁止重复触发，防止快速双击入口导致重复扣体力。
  - 缺失：点击入口后的扣体力原子性、扣体力失败后的页面回退规则未明确。

### 3.2 页面布局与移动端单屏系统

- **系统概述**: 负责无尽矿井在竖屏移动端的页面结构、信息展示和一屏内可操作性。
- **核心流程**:
  1. 玩家进入页面后，首屏同时看到顶部 HUD、状态信息区、顶部收获累计区、挖掘棋盘主体、右侧工具条、背包入口和「收队返场」。
  2. 页面视觉主体为中部 `7×6` 挖掘区域。
  3. 深度信息贴近棋盘中的深度线展示。
  4. 补给积分条位于挖掘区域正上方，作为轻量附属状态条。
  5. 高频工具操作优先采用右侧纵向独立工具条，并位于单手易点区域。
- **规则定义**:
  - 页面不依赖整体上下滚动查看核心玩法区域。
  - 棋盘、右侧工具条、底部背包与返场按钮必须在常见移动端竖屏尺寸下一屏完整呈现。
  - 棋盘必须横向居中，作为视觉焦点。
  - 可压缩非核心区域高度，但不得压缩棋盘到难以点击。
  - 优先缩减区域依次为：装饰性矿车/立牌、顶部信息卡高度、收获区留白、工具按钮外边距。
  - 不允许以缩小字体到难以识别的方式换取单屏。
- **资源关联**:
  - 顶部 HUD 展示主系统体力、金币、玉币。
  - 顶部收获累计区展示本局已获得金币、矿品/矿石、材料。
  - 底部或右侧工具区展示木镐子、横排炸弹、九宫格炸弹数量。
- **边界与异常**:
  - 窄屏下 `当前深度类型 / 本局推进 / 本局状态` 必须保持同一行展示，`收队返场` 允许独立占下一行。
  - 窄屏下 `金币 / 矿石 / 材料` 必须保持同一行展示，不允许自动拆成纵向三行。
  - 缺失：具体最小屏幕宽高、最小格子点击热区尺寸未明确。

### 3.3 顶部 HUD、状态信息与本局收获展示系统

- **系统概述**: 负责承载主系统资源显示、本局推进状态和本局收益累计感知，不承载高频操作。
- **核心流程**:
  1. 顶部 HUD 展示返回、体力、金币、玉币、设置。
  2. 状态信息区展示当前深度类型、本局推进、本局状态。
  3. 顶部收获展示区实时展示本局已获得金币、矿石/矿品、材料。
  4. 资源增长时做轻量数字跳动、闪光或飞入反馈，不弹出阻断式窗口。
- **规则定义**:
  - 返回按钮位于左上，点击后弹出确认框。
  - 顶部区域不放置高频点击道具按钮。
  - 宝箱不在顶部单独累计展示；开启后内容直接计入对应资源或工具数量。
  - 本局收获展示仅供查看，不作为点击入口。
  - 飞入特效目标锚点为顶部累计区，金币、矿石、材料三个槽位需各自接收对应资源飞入。
- **资源关联**:
  - 顶部 HUD 使用主系统体力、金币、玉币口径。
  - 本局收获在结算前累计展示，结算后回流主钱包、主背包/主仓库、主材料背包。
- **边界与异常**:
  - 顶部信息区中的数值卡、收获卡均应收紧高度，避免压缩棋盘可视面积。
  - 缺失：资源飞入动画队列拥堵、同时多资源掉落时的显示合并规则未明确。

### 3.4 矿井主场景与 7×6 挖掘区域系统

- **系统概述**: 负责展示矿井环境、可交互砖块区域、深度线和向下推进的空间感。
- **核心流程**:
  1. 页面中部展示“矿井环境 + 可交互砖块区域”。
  2. 当前可视采矿区域固定为 `7×6` 格。
  3. 玩家挖掘并清理砖块后，画面自动向下补充新砖块。
  4. 深度线展示在“当前采矿所在的最下方一行”和“倒数第二行”之间。
  5. 最下方一行始终保留，并展示真实的下一行矿井区域。
- **规则定义**:
  - 地图宽度固定 `7` 格，高度固定 `6` 格可视区域。
  - 地图向下无限生成，画面只展示当前附近可视区域。
  - 深度数值牌以“小牌子”形式贴在深度线侧边。
  - 最下方一行必须是真实下一行矿井区域，不是提示层、占位层或伪造预览层。
  - 棋盘内不应长期占用大块提示文案；规则应通过新手引导、轻量 toast 或首次提示完成。
- **资源关联**:
  - 挖掘区域产生矿品、金币、材料、积分、宝箱、工具补给。
  - 深度推进影响收益倍率、内容解锁、概率提升和地图生成难度。
- **边界与异常**:
  - 地图生成必须永远保证至少存在一条可继续向下挖的路径，避免纯随机封死。
  - 缺失：可继续下挖路径的判定算法、生成失败重试次数、可视区滚动/补行触发条件未明确。

### 3.5 资源模型与收益回流系统

- **系统概述**: 负责局内产出累计、结算汇总以及向主矿场资源体系回流。
- **核心流程**:
  1. 玩家挖掘砖块、宝箱或工具格获得局内产出。
  2. 矿品、金币、材料计入本局收获累计展示。
  3. 积分即时累计到当前补给积分 bar。
  4. 工具补给即时加入本局工具库存。
  5. 结算时本局收益统一回流到主矿场资源体系。
- **规则定义**:
  - 矿品价值统一参考既有矿品表，不在本模式内重新定义售价。
  - 本模式只影响矿品出现概率、数量和深度分布。
  - 收益统一进入主背包/主仓库、主钱包、主材料背包。
  - 宝箱被挖到后自动开启，其中矿石和工具数量直接叠加到当前本局收获与当前工具数量中。
  - 宝箱自动开启后，矿石和工具可以同时出现。
  - 掉落内容本身可以包含积分奖励，直接累加到当前积分 bar。
- **矿品表继承**:

| 矿石名称 | 稀有等级 |
| --- | --- |
| 普通石 | 1级·普通 |
| 萤石 | 2级·入门宝石 |
| 青金石 | 3级·稀有 |
| 紫晶 | 4级·珍稀 |
| 蓝晶 | 5级·高阶晶石 |
| 玉石 | 6级·贵重 |
| 火曜石 | 7级·活动稀有 |
| 陨铁 | 8级·传说 |
| 星辉石 | 9级·神话 |

- **边界与异常**:
  - 主背包满时，按主系统规则处理，不在本模式内单独兜底。
  - 结算中断线时，按服务端已记录收益补发。
  - 玩家退出游戏后，未结算收益按主系统规则恢复或结算。
  - 缺失：主系统背包满具体表现、补发通知文案、恢复或结算二选一的触发条件未明确。

### 3.6 砖块、难度实体与格子内容系统

- **系统概述**: 负责格子内容结构、砖块耐久、点击结果和掉落定位。
- **核心流程**:
  1. 地图生成浅红色沙砖、石砖矿、红砖盲盒矿、不可敲砖、宝箱、工具格等格子内容，并随机生成深红色墙面作为描述挖矿区域难度的实体。
  2. 深红色墙面不按大部分格子的基础内容生成；其在地图中随机出现，并希望出现位置具有连续性。
  3. 玩家用木镐子或炸弹破坏可破坏内容。
  4. 系统根据内容类型发放收益、工具或无效反馈。
- **规则定义**:

| 名称 | 耐久 | 说明 |
| --- | --- | --- |
| 深红色墙面 | 1 | 地图中随机出现且希望位置具有连续性的难度实体，与其他砖一样用于描述挖矿区域难度 |
| 浅红色沙砖 | 1 | 基础砖块，不挂钩收益，不直接产生基础金币和基础材料 |
| 石砖矿 | 2 | 高价值砖块，产出核心矿品 |
| 红砖盲盒矿 | 1 | 惊喜砖块，可能给宝箱或工具补给 |
| 不可敲砖 | 无限 | 障碍砖块，不可被普通挖掘破坏 |
| 宝箱 | 1 次发现 | 可直接出现在格子中，也可能在墙面下被发现，发现后自动开启 |
| 工具格 | 1 次发现 | 以 `🛠️` 形式出现在砖的正中间，相当于透视提示该格内有工具 |

| 内容 | 点击结果 | 掉落定位 |
| --- | --- | --- |
| 深红色墙面 | 敲击 1 次后碎掉 | 无直接收益 |
| 浅红色沙砖 | 直接破坏 | 无直接收益 |
| 石砖矿 | 先裂开再破坏 | 主矿场矿品、材料、少量炸弹 |
| 红砖盲盒矿 | 直接破坏 | 宝箱、工具补给、高波动收益 |
| 不可敲砖 | 无效反馈 | 无 |
| 宝箱 | 被挖到后自动开启 | 开出矿石或工具，并即时叠加 |
| 工具格 `🛠️` | 被挖到后直接获得工具 | 木镐子、炸弹等工具补给 |

- **资源关联**:
  - 浅红色沙砖不挂钩收益，不直接产生基础金币和基础材料。
  - 石砖矿产出主矿场矿品、材料、少量炸弹。
  - 红砖盲盒矿产出宝箱、工具补给、高波动收益。
  - 工具格产出木镐子、横排炸弹、九宫格炸弹。
- **边界与异常**:
  - 点击不可敲砖只给无效反馈，不消耗木镐子。
  - 炸弹只影响可破坏砖块，不影响不可敲砖。
  - PRD 内存在口径需关注：目录和 P0 写“5 类砖块”，6.1 表实际列出“深红色墙面、浅红色沙砖、石砖矿、红砖盲盒矿、不可敲砖、宝箱、工具格”等格子/内容项；其中深红色墙面按随机连续性难度实体处理，不作为基础格子内容；本文保持用户 PRD 全量口径，不删减。

### 3.7 深度分层、阶段推进与收益变化系统

- **系统概述**: 负责持续向下生成、深度类型切换、阶段节点变化、内容解锁与收益提升。
- **核心流程**:
  1. 玩家持续清理砖块，获得更深位置。
  2. 层数加深带来积分 +1，并推动补给积分条增长。
  3. 到达深度区间后切换当前深度类型。
  4. 达到更深阶段时弹出阶段提示。
  5. 深度越深，高价值矿砖占比、矿品数量倍率、盲盒矿收益上限、宝箱价值期望、稀有矿石出现概率、红砖盲盒矿出现概率、地图生成难度均提升。
- **规则定义**:

| 深度段 | 设计重点 |
| --- | --- |
| 地表矿区 | `1-30` 层，沙砖多、障碍少，帮助玩家消化开局 20 个木镐子的价值 |
| 深地表 | `31-80` 层，开始出现宝箱，仍以基础收益为主，地图开始更复杂 |
| 浅海 | `81-150` 层，从第三个深度类型开始出现稀有矿石和红砖盲盒矿，首次出现稀有矿石时给用户 toast 提示 |
| 深海 | `151-260` 层，红砖盲盒矿和宝箱在该深度类型内出现概率继续提升 |
| 地心火山泥 | `261+` 层，高价值资源最集中，但障碍、深红色墙面和高耐久砖占比最高 |

  - 每个深度类型包含层数不是等长，而是随着深度加深逐步变大。
  - 稀有矿石从第三个深度类型开始出现。
  - 宝箱从第二个深度类型开始出现。
  - 红砖盲盒矿从第三个深度类型开始出现。
  - 红砖盲盒矿和宝箱进入对应深度类型后即具有基础出现概率，并在每个深度类型中随深度继续加深逐步提高出现概率。
  - 矿石等级按层数逐步解锁：`1` 层可出现 `1级·普通` 和 `2级·入门宝石`；`2` 层可出现 `3级·稀有` 和 `4级·珍稀`；后续按相同规则递进。
  - 阶段信息区至少展示当前深度、当前深度类型、当前深度类型积分进度、下一阶段提示。
- **资源关联**:
  - 深度阶段影响高价值矿砖占比、矿品数量倍率、盲盒矿收益上限、宝箱价值期望、稀有矿石概率、红砖盲盒矿概率、补给积分门槛、补给奖励丰富度、地图生成难度。
- **边界与异常**:
  - 地图生成必须保证至少一条可继续向下挖路径。
  - 具体出现层数、概率曲线和阈值由数值表配置；用户 PRD 正文不写死。
  - 缺失：主数值表文件或具体数值配置未提供。

### 3.8 工具系统

- **系统概述**: 负责木镐子、横排炸弹、九宫格炸弹的库存、选择、释放、补给和消耗。
- **核心流程**:
  1. 开局发放 `20` 个木镐子。
  2. 玩家点击工具按钮切换当前工具，当前选中工具高亮。
  3. 木镐子点击可破坏砖块时消耗 `1` 个并造成 `1` 点伤害。
  4. 横排炸弹和九宫格炸弹先点击工具图标，再点击目标区域释放。
  5. 释放前显示范围预览，释放后破坏范围内可破坏砖块。
  6. 局内通过积分 bar、工具格、宝箱获得额外工具。
- **规则定义**:

| 工具 | 作用 | 来源 |
| --- | --- | --- |
| 木镐子 | 单体基础探索 | 每局固定 20 个开局 + 局中补给 |
| 横排炸弹 | 清理整行 | 局中补给 |
| 九宫格炸弹 | 清理局部范围 | 局中补给 |

  - 木镐子不继承上局剩余数量。
  - 点击可破坏砖块消耗 `1` 个木镐子并造成 `1` 点伤害。
  - 点击不可敲砖不消耗木镐子。
  - 炸弹不计入固定 `20` 个木镐子的基础配置中。
  - 横排炸弹直接破坏目标所在整行的可破坏砖块，不破坏不可敲砖。
  - 九宫格炸弹直接破坏目标周围九宫格内可破坏砖块，边界处只结算有效格子。
  - 若炸弹目标不可释放，给出无效提示。
  - 横排炸弹范围预览覆盖当前目标所在整行。
  - 九宫格炸弹范围预览覆盖目标周围 `3×3` 区域。
  - 范围预览与实际爆炸命中区域必须完全一致。
- **资源关联**:
  - 工具补给来源固定为 3 种：积分 bar 积满后获得固定补给工具；挖到工具格 `🛠️` 后直接获得工具；挖到宝箱 `🪎` 后自动开启并开出工具和矿石中的一种或多种组合。
  - 工具格当前只产出 3 种工具：木镐子、横排炸弹、九宫格炸弹。
- **边界与异常**:
  - 工具耗尽触发本局结束条件。
  - 缺失：横排炸弹和九宫格炸弹的初始数量、每次补给数量、补给概率、无效释放是否保持炸弹选中态未明确。

### 3.9 补给积分 bar 与局内补给系统

- **系统概述**: 负责通过深度推进和掉落积分累积进度，并在积满后发放工具补给。
- **核心流程**:
  1. 层数加深 `1` 层，积分 `+1`。
  2. 局内掉落内容可直接提供额外积分奖励 `+n`。
  3. 积分累计到当前深度类型的积分 bar。
  4. 积分 bar 积满后，获得固定补给工具。
  5. 深度增加后，积分 bar 积满门槛变大，奖励随深度类型提升更丰富。
- **规则定义**:
  - 积分 bar 的基础积分来源为“层数加深 `1` 层，积分 `+1`”。
  - 积分 bar 的积满所需敲击次数会随着深度逐步变大。
  - 积分 bar 积满后的奖励会随着深度类型提升而更丰富。
  - 奖励内容包含 `炸弹*n`、`木镐子*n`。
  - 每个深度类型中，都可能随机掉落当前深度类型最大层数 `30%` 的积分奖励。
- **资源关联**:
  - 积分即时累计到积分 bar，不作为结算资源回流主矿场。
  - 补给工具仅本局继续推进使用。
- **边界与异常**:
  - 缺失：各深度类型积分 bar 分母、固定补给工具种类和数量、积分溢出是否结转未明确。

### 3.10 结算、主动退出与返回确认系统

- **系统概述**: 负责主动止盈、工具耗尽结束、本局收获弹窗和返回矿区主页。
- **核心流程**:
  - 主动退出流程：
    1. 无尽矿井进行中。
    2. 玩家点击底部操作区最右下角独立按钮 `[收队返场]`。
    3. 当前收益停止增长。
    4. 汇总本局已获得资源。
    5. 弹出「本局收获」弹窗。
    6. 玩家查看本局深度和收成。
    7. 返回矿区主页。
  - 工具耗尽流程：
    1. 本局木镐子和炸弹全部耗尽。
    2. 本局立即结束。
    3. 进入本局收获结算弹窗。
    4. 本次挑战收益停止增长。
- **规则定义**:
  - 「收队返场」按钮需始终可见。
  - 「收队返场」固定放在最右下角，与常用工具按钮拉开间距，避免误触。
  - 「收队返场」不与镐子、炸弹按钮混排在连续点击热区中。
  - 点击 `收队返场` 后直接进入结算弹窗。
  - 工具耗尽后直接进入结算弹窗。
  - 左上返回按钮点击后弹出确认框：
    - 标题：`确定收队返场？`
    - 内容：`当前已获得的收益会立即结算并带回矿区。`
    - 按钮：`继续下挖` / `确认返场`
- **资源关联**:
  - 结算汇总矿品、金币、材料等本局收益，并回流主矿场。
  - 工具补给仅在本局使用，结算时不作为跨局继承资源。
- **边界与异常**:
  - 结算中断线按服务端已记录收益补发。
  - 玩家退出游戏后，未结算收益按主系统规则恢复或结算。
  - 缺失：结算弹窗字段、确认返场后按钮防重、结算失败重试规则未明确。

### 4.2 资源流转

```text
主矿场体力 -1
    ↓
进入 1 局无尽矿井
    ↓
系统发放本局固定 20 木镐子
    ↓
点击砖块 / 使用横排炸弹 / 使用九宫格炸弹
    ↓
获得矿品、金币、材料、积分、宝箱、工具补给
    ↓
积分进入本局补给积分 bar；工具补给仅本局继续使用
    ↓
主动收队返场或工具全部耗尽
    ↓
本局收获结算
    ↓
矿品进入主背包/主仓库；金币进入主钱包；材料进入主材料背包
```

- **已明确闭环**:
  - 体力限制开局次数。
  - 木镐子定义本局基础探索次数。
  - 深度决定本局收益上限。
  - 收益统一回流主矿场资源体系。
- **已明确验算**:
  - `1` 点体力可开启 `1` 局。
  - `1` 局开局固定 `20` 木镐子。
  - 使用木镐点击可破坏砖块：每次消耗 `1` 木镐并造成 `1` 点伤害。
  - 若没有任何局内补给，基础木镐可支持最多 `20` 次木镐消耗行为；实际可破坏内容数受石砖矿 `2` 耐久、深红色墙面、炸弹补给、工具格和宝箱补给影响。
- **缺失**:
  - 无法完成收益期望验算：各砖块概率、掉落数量、收益倍率、宝箱价值期望未提供。
  - 无法完成工具续航期望验算：工具格概率、宝箱工具概率、积分 bar 门槛和奖励数量未提供。

### 4.3 成长曲线

| 阶段 | 层数 | 已明确内容 | 缺失信息 |
|------|------|------------|----------|

| 深地表 | `31-80` | 开始出现宝箱，仍以基础收益为主，地图开始更复杂。 | 宝箱基础概率、宝箱奖励表、复杂度增长参数。 |

## 五、核心状态机

### 5.1 状态枚举

| 状态 | 含义 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| `mine_home_available` | 矿区主页中无尽矿井入口可进入 | 玩家在矿区主页，体力满足进入条件 | 点击无尽矿井入口并通过防抖进入入场处理 |
| `mine_home_energy_insufficient` | 矿区主页中无尽矿井入口不可进入 | 玩家体力不足 | 体力恢复或获得到可进入数量 |
| `entering_mine` | 入场处理 | 点击无尽矿井入口且体力足够；入场处理中入口防抖，禁止重复触发 | 扣除体力并发放本局 `20` 木镐后进入推进中；缺失：扣体力失败状态未明确 |
| `progressing` | 常规推进状态 | 入场完成，或从阶段提示/工具选择/确认弹窗返回继续下挖 | 点击砖块、选择工具、释放炸弹、阶段提升、点击收队返场、点击左上返回、工具全部耗尽 |
| `tool_selected` | 已选中某个工具 | 玩家点击木镐子、横排炸弹或九宫格炸弹工具按钮 | 对目标格释放、切换工具、取消选择；缺失：取消选择规则未明确 |
| `bomb_preview` | 炸弹释放预览 | 玩家选中横排炸弹或九宫格炸弹并指向目标区域 | 确认释放、目标不可释放提示、切换工具、退出；缺失：具体取消手势未明确 |
| `dig_resolving` | 单次挖掘/爆炸结算中 | 玩家点击可破坏砖块或释放炸弹 | 破坏、掉落、飞入、工具消耗、深度/积分变化结算完成 |
| `stage_up_prompt` | 阶段提升提示 | 达到更深阶段或切换深度类型 | 提示展示完成后回到推进中 |
| `rare_unlock_toast` | 首次解锁提示 | 稀有矿石、红砖盲盒矿或宝箱首次出现 | toast 结束后回到推进中 |
| `return_confirm` | 返回确认弹窗 | 玩家点击左上返回按钮 | 点击 `继续下挖` 回到推进中；点击 `确认返场` 进入结算中 |
| `manual_exit_requested` | 主动收队返场请求 | 玩家点击 `收队返场` | 当前收益停止增长，进入结算中 |
| `tools_depleted` | 工具耗尽状态 | 本局木镐子和炸弹全部耗尽 | 展示 `工具已耗尽` / `本局挖掘结束，开始结算收获` 后进入结算中 |
| `settling` | 结算中 | 主动退出、确认返场或工具耗尽 | 汇总收益完成后展示本局收获弹窗；断线时按服务端已记录收益补发 |
| `result_popup` | 本局收获弹窗 | 结算完成 | 玩家查看本局深度和收成后返回矿区主页 |
| `returned_to_mine_home` | 返回矿区主页 | 玩家从本局收获弹窗确认返回 | 结束本次无尽矿井流程 |
| `interrupted` | 异常中断 | 结算中断线或玩家退出游戏 | 按服务端已记录收益补发，或未结算收益按主系统规则恢复或结算；缺失：恢复/结算条件未明确 |

## 八、验证重点与验收口径

### 8.1 玩家是否理解单局成本

重点验证玩家是否能快速理解：

- 进入一次要消耗 `1` 点体力。
- 每局开局固定只有 `20` 个木镐子。

### 8.2 玩家是否理解深挖收益

重点验证玩家是否能感知：

- 越深层越值钱。
- 同样 `1` 个木镐子在深层更有收益价值。
- 达到更深层后会逐步解锁稀有矿石、红砖盲盒矿和宝箱。
- 进度条奖励虽然更难积满，但奖励也更丰富。
- 地图虽然更难，但始终存在至少一条可继续下挖路径。

### 8.3 工具耗尽后的结束反馈是否成立

重点验证玩家是否会在：

- 主动止盈
- 工具耗尽结算

之间形成清晰认知，而不是对结束时机感到困惑。

### 8.4 单局时长是否适合移动端

重点验证：

- 固定 `20` 个木镐子开局是否让节奏足够紧凑。
- 在当前固定资源规则下是否仍适合竖屏移动端短时体验。
