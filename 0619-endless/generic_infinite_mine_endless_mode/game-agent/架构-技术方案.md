### API 设计

| 接口 | 函数名 | 参数 | 返回 | 说明 |
|------|--------|------|------|------|
| 获取矿区主页 | `getMineHome` | — | `ApiResponse<MineHomeData>` | 获取体力、金币、玉币、入口可用态 |
| 进入无尽矿井 | `submitEnterEndlessMine` | `{ costEnergy: 1 }` | `ApiResponse<EnterMineData>` | 校验体力、扣除 1 点体力、创建局、发放 20 木镐 |

| 提交结算 | `submitSettleMine` | `{ runId; reason; clientSnapshot }` | `ApiResponse<SettlementData>` | 主动退出、返回确认或工具耗尽时结算 |
| 查询未结算局 | `getPendingMineRun` | — | `ApiResponse<PendingRunData | null>` | 中断恢复/补发兜底 |

## Hook 设计

### useMineHome()

- **输出**: `{ homeState, handleEnterMine(), handleCloseBlocker() }`
- **内部状态**: `entryLockRef`（入口防重锁）
- **核心逻辑**:
  1. mount 调用 `getMineHome()` 写入主系统资源。
  2. 点击入口时检查 `homeEnergy >= 1`，不足则打开 `ModalBlocker` 或 toast。
  3. 体力足够时设置防重锁，调用 `submitEnterEndlessMine()`。
  4. 成功后初始化 `mine` slice，切换 `route=endless_mine`。
  5. `finally` 释放防重锁。

### useEndlessMine()

- **输出**: `{ canvasHostRef, handleSelectTool(), handleExit(), handleReturnClick(), handleConfirmReturn(), handleContinueDig(), handleResultConfirm() }`
- **内部状态**: `gameRef`、`settleLockRef`、`actionLockRef`、`harvestDomRefs`
- **核心逻辑**:
  1. mount 后创建 `EndlessMineGame`，加载配置与素材，注册强类型 callbacks。
  2. callbacks 中 dispatch 棋盘、工具、深度、补给、收获、toast、状态变化。
  3. 工具选择和 Canvas 点击通过 `game.selectTool()`、`game.digCell()`、`game.useBomb()` 完成。
  4. 主动 `收队返场` 直接进入结算；左上返回先打开 `ModalConfirmReturn`。
  5. 工具耗尽回调触发 `submitSettleMine()` 并打开 `ModalResult`。
  6. unmount 调用 `game.destroy()`，清理 Pixi Application、Ticker、纹理引用、事件监听。

## Mock 数据方案

- Mock 函数使用闭包维护状态：体力扣减、runId、自增 clientSeq、局内工具和收益快照。
- 返回新对象（展开运算符）保证 React 重渲染。
- 模拟 150-300ms 网络延迟。
- 通过 `import.meta.env.VITE_USE_MOCK` 切换 Mock/真实 API。
- **Mock 本地权威 vs 真实 API patch 分流**：Mock 模式下，`submitDigAction()` 仅作为挖掘上报与网络延迟占位，不把返回值作为权威修正；UI 以本地 `EndlessMineGame` 已完成的棋盘、收益、补给和工具演算结果为准。仅在真实 API 模式下，`submitDigAction()` 返回的 `DigServerPatch` 才作为 server authoritative correction，由 `useEndlessMine` 调用 `game.applyServerPatch(patch)` 并同步 Redux 快照。
- 缺失数值（砖块概率、补给门槛、工具补给数量、掉落概率）在 Mock 中仅作为演示配置，不写入 PRD 语义；正式环境由 `getEndlessMineConfig` 返回配置覆盖。

## 性能约束

| 指标 | 目标 | 实现手段 |
|------|------|---------|
| 首屏加载 | < 3s | Pixi CDN 外链、素材按场景预加载、入口页先渲染 DOM |
| JS bundle | < 200KB gzip | Pixi 不进业务 bundle；只打包 React/Redux/业务代码 |
| 图片资源 | < 1MB | 元素清单初判合并背景/复用素材；格子与图标用透明 PNG 复用 |
| 动画帧率 | ≥ 30fps | Canvas 高度控制在屏幕 55% 左右；Ticker 只在玩法页运行；粒子数量限流 |
| 内存泄漏 | 0 | `destroy()` 清理 Pixi、ticker、事件、setTimeout；React unmount 清理 Hook |
| Canvas 高度 | ≤ 屏幕 50-60% | 中部 MineScene 为唯一 Pixi Canvas，顶部/底部/弹窗使用 React DOM |
| 交互响应 | 点击后 100ms 内反馈 | 本地先播放命中反馈，服务端 patch 后校正收益 |

| Slice | 字段 | 读/写 | 触发时机 |
|-------|------|-------|---------|
| app | `homeEnergy/homeCoin/premiumCurrency` | R+W | `getMineHome` 成功 |

| app | `route` | W | 入场成功切换玩法页 |

#### 涉及 Hooks

| Hook | 用途 | 关键输出 |
|------|------|---------|
| `useMineHome` | 主页数据加载和入场防抖 | `handleEnterMine` |

#### 涉及 API

| API | 调用时机 | 说明 |
|-----|---------|------|
| `getMineHome` | 页面 mount | 拉主系统资源和入口状态 |
| `submitEnterEndlessMine` | 点击入口且体力足够 | 扣 1 体力、创建局、发 20 木镐 |

#### 核心数据流（时序）

```
1. MineHomePage mount → useMineHome → getMineHome → updateApp(resource)
2. 点击无尽矿井入口 → entryLockRef 检查
3. 体力不足 → openModal(blocker: 体力不足)
4. 体力足够 → submitEnterEndlessMine → updateMine(runId/tools=20 pickaxe) → setRoute(endless_mine)
```

#### 页面内开发顺序

1. 搭建 TopHud / HomeScene / EndlessEntryCard / BottomDock DOM 结构。
2. 完成 appSlice 与 useMineHome。
3. 接入 Mock API 和入口防抖。
4. 接入 ModalBlocker。
5. 联调入场后 mineSlice 初始化。

#### 涉及 API

| API | 调用时机 | 说明 |
|-----|---------|------|

#### 核心数据流（时序）

```
1. EndlessMinePage mount → useEndlessMine → getEndlessMineConfig
2. new EndlessMineGame().init(canvasHost, config, callbacks) → game.start(runData)
3. 点击工具 → handleSelectTool → game.selectTool → callbacks.onToolChange → dispatch
4. 点击格子 → game.digCell/useBomb → Pixi 播放反馈 → 本地结算/submitDigAction → callbacks.onHarvestChange/onBoardChange/onDepthChange
5. 资源飞入 → EffectLayer.flyResource(fromGrid, RunHarvest chip DOMRect) → callbacks.onHarvestChange → RunHarvest 数值跳动
6. 点击收队返场 → handleExit → submitSettleMine → setSettlement → openModal(result)
7. 点击左上返回 → openModal(confirm_return) → 继续下挖 closeModal / 确认返场 submitSettleMine
8. 工具耗尽 → callbacks.onGameOver(tools_depleted) → submitSettleMine → openModal(result)
9. 结果确认 → resetMine → setRoute(mine_home)
```
