### 后台 API
> 工程层 API 字典：每个 API 提供 **JCE 类型坐标**（模块 + Req/Rsp 符号）+ 请求路径 + 错误处理。
> `Req`/`Rsp` 列是 JCE 声明里的符号名，coding 据此从 `JCE 模块` 列 re-export，**禁止照字段手写 interface**。
> 「哪个元素调用哪个 API」的承接关系由元素清单 `api` 列负责（值即 API 名），本表不写“关联功能”列。

| API 名 | 请求路径/cmd | JCE 模块 | Req 类型 | Rsp 类型 | 错误处理 |
|---|---|---|---|---|---|
| queryActivityStatus | kg.aigc_sticker_gift.webapp.query_activity_status | jce/kg/aigc_sticker_gift/proto_aigc_sticker_gift_webapp | $QueryActivityStatusReq | $QueryActivityStatusRsp | 进入页加载失败时展示兜底提示；生成状态刷新失败时保持当前页面状态并允许重试 |
| submitAuth | kg.aigc_sticker_gift.webapp.submit_auth | jce/kg/aigc_sticker_gift/proto_aigc_sticker_gift_webapp | $SubmitAuthReq | $SubmitAuthRsp | 透传后台错误信息；重复提交或能力异常时解除本地提交中状态并保留当前选择 |

### ⚠️ JCE 合理性检查
> 基于 PRD 业务语义的轻量检查，仅供参考；如发现问题请与后台同学确认是否调整 JCE。

#### 字段缺失
- `$SubmitAuthReq` 仅有 `strImageUrl`：PRD F003 支持上传 1-4 张照片，若后台要求多图生产需确认是否支持传多图或由上传链路先聚合为单个生产图 URL。
- `$QueryActivityStatusRsp` 缺少礼物名称、主播昵称、表情包分类名称、表情包 icon、礼物价值等字段：PRD F011 需要展示生成后的表情包礼物信息。

#### 类型合理性
- `$QueryActivityStatusRsp.emAuthStatus` 使用 `emActivityStatus`：PRD D001/D005 区分授权状态与生成状态，需确认是否应使用独立授权/生成状态枚举以避免语义混淆。

## 5. 全局状态字典
> 跨页面共享的核心状态，提供字段类型。
> 「哪个元素读写哪个字段」的承接关系由 §2.1 路由元数据按路由粗粒度聚合。

| 字段 | 类型 | 业务含义 | 关联 PRD D### |
|---|---|---|---|
| authStatus | 'unauthorized' \| 'submitted' \| 'completed' | 主播授权与生产流程状态 | D001 |
| imageSource | 'anchorAvatar' \| 'uploadedPhoto' | 本次 AI 生产使用默认头像或上传照片 | D002 |
| uploadedPhotos | string[] | 主播上传并可用于 AI 生产的照片集合 | D003 |
| agreementChecked | boolean | 主播是否同意授权须知 | D004 |
| generationStatus | 'idle' \| 'generating' \| 'completed' \| 'failed' \| 'timeout' | AI 表情包礼物生产进度 | D005 |
| participation | { canAuth: boolean; limitReason?: number; limitText?: string; limitToastText?: string } | 主播是否满足等级限制或白名单限制 | D006 |
| pageConfig | { backgroundImage?: string; sampleImageUrl?: string; buttonStyle?: object; agreement?: object; uploadArea?: object } | 授权页面展示所需的可配置内容 | D007 |
| giftInfo | { giftName?: string; icon?: string; price?: number; categoryName?: string; anchorNick?: string } | 生成后的礼物展示所需信息 | D008 |
