## 需求正文

**一、需求背景**

用户存在情感表达与社交互动需求，以礼物形式验证「表情包」互动方式收益高与日常礼物，调研反馈用户更倾向在日常互动聊天中送表情包礼物，所以将表情包能力升级为守护特权，强化主播与用户之间的1v1亲密关系

历史数据：上线4款宇仲表情包，日均售卖2167元占宇仲日流水13.7%，同档位礼物日均近似价位礼物日均售卖62.5元；对比活动上线前一周，近似档位礼物日均有明显提升，先抽出作为用户权益下发

**1、授权能力**

* 通用能力，支持不同玩法调用/关联该授权能力，主播授权后其他玩法才可进行后续生产

**2、授权流程：**

* 授权时机：
  + 主播开播前：运营banner推广，点击拉起授权页面——运营配置，无需开发
  + 主播开播时：IM消息提醒主播：进行授权，点击拉起授权页面——IM消息文案支持配置
  + 主播下播时：运营banner推广，点进拉起授权页面——运营配置，无需开发
  + 私信时机：——待确定是调用平台能力（配置图片+文案=跳转活动页面），还是玩法内配置
* 授权流程：
* ![](/tfl/captures/2026-06/tapd_10088931_base64_1780544465_293.png)

* + 主播通过私信或者挂件、资源位点击拉起授权页面，在页面进行授权，授权选择2种
    - **拉取主播头像**：默认展示拉取的主播头像，默认勾选状态。若主播不更改选择，点击「立即生成」，则以主播头像进行AI生产
    - **重新上传照片：**支持主播上传1-4张照片，主播勾选照片选择下一步后，回到主页面。页面照片选择框默认展示第一张照片，默认勾选状态。若主播不更改选择，点击「立即生成」，则以主播上传照片进行AI生产
      * 支持主播重新更换照片
    - 主播可切换选择用默认头像或者用上传的照片（二选一）进行AI生产，一旦点击「立即生成」则照片不可更改，不可二次生成
    - 同意《AI表情包授权须知》：默认为勾选状态
  + 主播点击立即生成后，按钮状态改为预估生成时间「预估x分后生成完成」
    - 刷新：进入页面时刷新、停留在页面时若生成成功则页面刷新，按钮文案改为「生成完成｜
  + 授权仅1次，页面需要说明「授权官方使用该图片进行表情包生产」
  + 页面支持配置内容
    - 支持配置的主题文案图片
    - 支持配置页面预览图片（支持配置动图，可自动循环展示动画）
    - 支持配置页面皮肤颜色
    - 支持配置挂件背景、及点击按钮颜色
* 限制条件：
  + 等级限制&主播白名单支持二选一配置，若配置为
    - 主播等级限制：则不满足主播等级条件的主播，不可参与生产，页面按钮状态变为「不满足条件」，点击toast「升级主播等级可解锁该权益」——支持运营配置主播等级
  + 支持白名单主播参与
    - 页面及私信仅投放白名单主播
* 授权结果：
  + 主播授权生成对应表情包礼物后，底部弹窗提示（主播/房间用户均可见）：xxx（主播昵称）主播的xxx表情包礼物已上线，快去查看（按钮-点击拉起礼物面板）
  + 礼物面板均展示已经生成的表情包，支持用户消费送出

**2、表情包礼物入口**：

* 生成的AI表情包礼物：默认进入专属礼物tab顶部展示，展示「第一个表情包icon+名称+价值」
* 表情包礼物名称：以主播昵称+表情包分类名称组合，比如丛书的守护专属、丛书的美食系列等
