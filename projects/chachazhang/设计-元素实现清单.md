> - **列值规则**：
>   - `img` 列：snake_case 资源 key（如 `bg_home`、`btn_start`）；该 key 直接作为 `images.ts` 字段名；review 标 `Y` 的元素必须在本阶段命名为具体 key；review 标 `—` 的保持 `—`
>   - `api` 列：后台 API 函数名（如 `fetchRoundInfo` / `submitChoice`），多个用 ` + ` 分隔；该 API 必须在技术方案 §4 字典中存在；无则 `—`。判定标准：仅当元素触发或承接后台 API 时填写
>   - `gamesdk` 列：GameSDK 方法全名（如 `tmegame.user.getUserInfo`），多个用 ` + ` 分隔；无则 `—`。判定标准：仅当元素需要调用 GameSDK 能力时填写；**后台 API 不属于 gamesdk**（走 api 列）
>   - `component` 列：组件引入路径（`library/components/AdvancedImage(type='avatar')`）或章节锚（`common-components.md#Marquee-跑马灯`）；无则 `—`
>   - `skill` 列：Skill 全限定名（如 `gamesdk-gift-kg:gift-send-panel`），多 Skill 用 ` + ` 串联；无则 `—`
>   - `fx` 列：动效 ID（如 `FX01`），多个用 ` + ` 分隔；该 FX 必须在技术方案 §8 字典中存在；无则 `—`
>   - `复用标注` 列：review 初稿字面值（`复用源` / `复用 E##` / `—`）一字不改保留；**同一复用组（首现 `复用源` + 后续 `复用 E##` 指向它）的元素，`component` 列必须填同一值**（跨场景共用同一组件）
> - **六维度互斥规则**：`skill ≠ —` 的元素，同行 `gamesdk` / `component` 必须为 `—`（Skill 内部已封装对应能力）；例外仅当 Skill 明确不承接该能力时，可串联或下沉，并在 `元素描述` 列写明理由
> - **元素描述列**承接业务条件、状态分支、注意事项、文案 T### 引用、装饰用途、D### 数据来源、Skill 串联或例外理由等所有补充信息；但禁止重复其他列里已写明的资源 key / API 名 / SDK 名 / 组件路径 / Skill 名 / FX###；纯装饰元素（关联功能ID = `—（装饰）`）的「元素描述」必须说明装饰用途
