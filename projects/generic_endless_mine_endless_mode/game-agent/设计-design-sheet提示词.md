📋 🇨🇳:
生成一张横向 3:2 画板，展示《无限矿井》无尽模式 3 个核心场景的 UI 缩略图；缩略图外区域是纯绿色 #00FF00 画板背景，缩略图内部是正式游戏 UI 成片截图。

【单行排布】
3 个严格 9:16 竖屏缩略图在 3:2 画板上单行横向排列，每个缩略图上下左右留清晰的纯绿 #00FF00 间隙。各格尺寸一致、水平等距、垂直居中。宁可格子小一些、绿边多一些，也严禁为填满高度或宽度把缩略图拉高、压宽、贴边；不要让任何缩略图变成横向或非 9:16 单元格。

【核心场景清单】
（1）标题页默认态（TitlePage / title）——整屏标题背景承载矿井入口场景，画面上部展示游戏标题艺术字，中下部留出进入无尽矿井的主按钮，作为从预加载后进入唯一可玩关卡的入口页。
（2）无尽矿井主游玩界面默认态（RunPage / endless_run）——竖屏移动端单屏承载顶部状态行、顶部收益行、7×6 矿井棋盘、深度线、深度牌、补给条和底部工具区；底部工具区包含镐子、横炸、九炸、背包、收队返场五个入口；基础页面缩略图内不绘制任何弹窗、遮罩或 toast。
（3）本局收获结算页默认态（SettlementPage / settlement）——主动返场或工具耗尽后展示本局最终数据；中部结果面板列出最终深度、深度类型、金币、矿石、材料和本局开启宝箱次数，底部展示再来一局与返回标题两个 Demo 导航按钮。

【跨页统一】
这 3 个核心场景共同覆盖全部跨页共享元素：地表矿区背景、标题与结算页的主标题视觉身份、主行动按钮体系、结果面板与工具底板的卡框体系、资源图标体系、矿井棋盘格与提示物、工具按钮槽、木镐图标、横排炸弹图标、九宫格炸弹图标、背包按钮、收队返场按钮、再来一局按钮、返回标题按钮。同一共享元素在所有缩略图中保持相同造型、色相、切面结构、笔触和材质；按钮体系、卡框体系、资源图标、工具图标、棋盘格、背景变体的视觉语言必须跨缩略图一致。

【质量要求】
- 每个缩略图都是高完成度 UI 设计稿，像商业上线项目截图
- 画面干净，主体突出，信息清晰
- 缩略图内部为正式游戏 UI 成片截图，外部仅为纯绿色画板背景
- 分辨率：2K
- 画布比例：3:2，3072×2048px

【负面约束】
- 不要 annotation，不要 label，不要 callout，不要 redline，不要 measurement line
- 不要 blueprint，不要 wireframe，不要 technical notes，不要坐标，不要尺寸，不要编号
- 不要边界框，不要辅助线，不要标尺，不要指示箭头
- 不要在画面上出现文件名、变量名、英文占位符
- 基础页面缩略图内不绘制任何弹窗、遮罩、toast、跑马灯或结算浮层
- 不要把缩略图拉伸成横向，不要让 UI 填满非 9:16 单元格；每个缩略图保持 9:16 竖屏、四周留绿

- 当前深度文本（层数显示）：位于深度牌容器内，展示当前层数或深度进展。
- 补给条容器（补给积分面板）：位于棋盘上方的信息栈中，与深度牌共同构成棋盘上方的深度补给区域；补给进度条横向展开。
- 补给进度填充（补给进度可视化）：位于补给条容器内，表示 supplyPoints 到 supplyTarget 的当前进度。
- 补给进度文本（补给数值）：位于补给条容器内，展示本局推进积分与目标。
- 补给内容展示（补给奖励预告）：位于补给进度条最右侧，展示本次补给内容，例如“补给：木镐+4”或对应深度区工具补给；该信息必须和进度条同一行右端对齐，不能放到棋盘下方或底部工具栏。
- Pixi 棋盘视口容器（中部主玩法区）：位于深度/补给信息下方、底部工具栏上方，是 7×6 矿井棋盘的主操作区域，视觉层级高于背景且低于顶部和底部 UI。

- 九宫格炸弹图标（工具图标）：位于九炸工具按钮内，配合工具数量文本表示九宫格炸弹库存。
