## 技术栈

| 类别 | 选型 | 版本 | 说明 |
|------|------|------|------|
| 框架 | React | 18.x | UI 层、页面状态、HUD、结算页 |
| 渲染引擎 | Canvas 2D（standalone 等价实现） | Browser Canvas API | 本轮代码实现采用 Canvas 2D 渲染 7×6 棋盘与轻量反馈，保留 `MineBoardRenderer` 接口语义；与原 Pixi 方案差异为不创建 Pixi Application/Sprite/Container，避免额外运行时依赖，不改变棋盘逻辑、资源语义或验收可观察结果 |
| 语言 | TypeScript | 5.x | 全局类型、Store、引擎回调强类型 |
| 样式 | Less + postcss-pxtorem | — | 750px 移动端设计稿适配，页面主比例为竖屏 9:16 |
| 状态管理 | Redux (@reduxjs/toolkit) | 2.x | 保存跨组件运行态、HUD 和结算数据 |
| 构建 | Vite | 5.x | `npm run dev` 即可预览 |

> 渲染实现同步说明：现有 standalone 代码以 Canvas 2D 作为原 Pixi 方案的等价实现；若后续切回 Pixi，可复用 `EndlessMineGame`、`MineBoardModel`、`MineRowGenerator`、`RewardResolver` 与 `AudioGenerator` 的接口边界。

## 引擎核心类设计（Canvas 2D standalone 等价实现）

### EndlessMineGame

```typescript
class EndlessMineGame {
  init(canvas: HTMLCanvasElement, config: MineGameConfig, callbacks: MineGameCallbacks): Promise<void>;
  startRun(initialState: MineRunState): void;
  selectTool(tool: ToolType): void;
  handleCellClick(gridX: number, gridY: number): void;
  requestReturnToCamp(): void;
  resize(width: number, height: number): void;
  destroy(): void;
}
```

职责：维护 Canvas 2D 棋盘、程序化音频、输入防抖、处理流水线调度、结算触发；通过 Typed Callbacks 回写 Redux，不引入 EventBus。Canvas 挂载在 RunPage 中部 `GameViewport`，按竖屏容器缩放 7×6 棋盘。

### MineBoardModel

```typescript
class MineBoardModel {
  terrain: CellType[][];
  metadata: MineCellMeta[][];
  isLegalTarget(x: number, y: number, tool: ToolType): boolean;
  applyDamage(targets: GridPoint[], damage: number): ProcessResult;
  canAdvanceFromBottom(): boolean;
  rollOneRow(newRow: GeneratedRow): void;
  getSnapshot(): BoardSnapshot;
}
```

职责：封装 7×6 棋盘、四方向连通、WALL 不可破坏、遮挡层优先破坏、底部通路检测。

### MineBoardRenderer

```typescript
class MineBoardRenderer {
  loadAssets(assetManifest: MineAssetManifest): Promise<void>;
  mount(stage: PIXI.Container): void;
  renderBoard(snapshot: BoardSnapshot): void;
  playHit(point: GridPoint): Promise<void>;
  playBreak(points: GridPoint[]): Promise<void>;
  playBomb(type: 'rowBomb' | 'areaBomb', targets: GridPoint[]): Promise<void>;
  playRewardFly(reward: RewardPayload): Promise<void>;
  playRoll(): Promise<void>;
  resizeToViewport(width: number, height: number): void;
  destroy(): void;

```

职责：只处理 Canvas 2D 图片绘制、点击反馈与破坏反馈；不直接修改 Redux。`resizeToViewport` 以竖屏单屏中部棋盘容器为边界，优先保证 7 列可读、6 行完整可见。

### MineRowGenerator / RewardResolver

```typescript
class MineRowGenerator {
  generate(depth: number, biomeIndex: number, currentBottomOpenings: number[]): GeneratedRow;

class RewardResolver {
  resolve(meta: MineCellMeta, biomeIndex: number): RewardPayload;
```

职责：生成新底行和结算奖励。PRD 缺失概率表时，Mock 配置采用可替换的本地占位表，并在常量中标注来源为 Demo fallback。

### AudioGenerator

```typescript
class AudioGenerator {
  register(): Record<AudioCue, string>;
  playMineHit(): void;
  playMineBreak(): void;
```

职责：当前项目无真实音频素材时，使用 Web Audio 程序化生成等价 mock audio，并注册 `mine_bgm`、`mine_hit_sfx`、`mine_break_sfx`、`mine_bomb_row_sfx`、`mine_bomb_area_sfx`、`mine_reward_sfx`、`mine_depth_up_sfx`。音频入口由 `EndlessMineGame` 在初始化、合法挖掘、砖块破坏、炸弹、奖励入账和深度提升时调用，且不修改玩法数据。

## Hook 设计

### useEndlessMineGame()

- **输出**: `{ canvasRef, startRun(), selectTool(tool), returnToCamp(), resizeGame() }`
- **内部状态**: `gameRef`（Canvas 2D 游戏实例）、`lockRef`（异步防重锁）、`settlementTimerRef`、`latestRunStateRef`
- **核心逻辑**:
  1. `useEffect` 创建 `EndlessMineGame`，传入 canvas、配置和 typed callbacks。
  2. 根据 `GameViewport` ResizeObserver 触发 `resizeGame()`，保证竖屏移动端单屏内棋盘完整可见。
  3. `callbacks.onStatePatch` dispatch `updateRun`。
  4. `callbacks.onToast` dispatch `showToast`。
  5. `callbacks.onSettle` 写入 `finalSnapshot`，等待 `600ms` 后切到 `settlement`。
  6. unmount 时清理 Canvas 2D、timer、监听器，保证 0 内存泄漏。

### useAppActions()

- **输出**: `{ goTitle(), startNewRun(), replay(), submitSettlementAndShow() }`
- **内部状态**: `lockRef`
- **核心逻辑**: 封装 `getHomePage`、`submitStartRun`、`submitRunSettlement` 调用与路由切换。

### useRouteGuard()

- **输出**: `{ canInteract }`
- **内部状态**: 无额外状态
- **核心逻辑**: 根据 `route`、`inputLocked`、`runStatus` 判断 DOM 按钮是否可点击。

## Mock 数据方案

- Mock 函数使用闭包维护状态：体力展示值、最近一局结算结果、累计开始次数。
- 返回新对象（展开运算符）保证 React 重渲染。
- 模拟 150-300ms 网络延迟。
- 通过 `import.meta.env.VITE_USE_MOCK` 切换。
- 缺失的掉落概率、宝箱/盲盒表以 `rewardConfig.ts` 中 `demoFallback` 字段隔离，便于后续替换为正式 PRD 表。

## 性能约束

| 指标 | 目标 | 实现手段 |
|------|------|---------|
| 首屏加载 | < 3s | Canvas 2D CDN 外链；首屏仅预加载标题背景、当前 biome 背景、tileset 和 HUD 图标 |
| JS bundle | < 200KB gzip | Canvas 2D 不打入 bundle；按页面拆分组件；移除未用依赖 |
| 图片资源 | < 1MB 初始关键路径 | 深层背景按 biome 延迟加载；标题和地表资源优先 |
| 动画帧率 | ≥ 30fps | 棋盘 Sprite 复用；避免每帧创建对象；资源飞字结束后归还对象池 |
| 内存泄漏 | 0 | `destroy()` 清 Canvas 2D application、ticker、texture 引用、DOM listener、timer |
| Canvas 高度 | 目标 ≤ 屏幕 60% | RunPage 使用纵向堆叠，Canvas 2D viewport 位于顶部 HUD 与底部工具区之间，7×6 棋盘按容器等比缩放 |
| 输入防抖 | 120ms | 引擎层记录 `lastActionAt`，动作处理中锁输入 |

```text
4. 用户点击格子 -> EndlessMineGame.handleCellClick(gridX, gridY)
5. 校验工具库存、连通性、WALL、遮挡层 -> 合法则进入处理流水线
6. MineBoardModel.applyDamage -> RewardResolver.resolve -> MineBoardRenderer 播放反馈
7. callbacks.onStatePatch / onLootChange -> dispatch(updateRun / addLoot)
8. 底部通路成立 -> MineRowGenerator.generate -> rollOneRow -> currentDepth + 1
9. 三类工具均 <= 0 或点击收队返场 -> callbacks.onSettle
10. 600ms 结算延迟 -> submitRunSettlement -> route = settlement
```

#### 页面内开发顺序

1. 实现 `MineRunState`、常量配置、初始棋盘数据。
2. 实现 RunPage 竖屏移动端单屏 DOM 框架和 HUD 组件。
3. 实现 Canvas 2D `EndlessMineGame` 初始化与资产预加载。
4. 实现 `MineBoardModel` 合法性、破坏和推进。
5. 接入工具栏、奖励、toast 和结算流。
6. 补齐动画与音频反馈。
